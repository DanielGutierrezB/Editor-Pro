# Smart Supertext MOGRTs — Technical Reference

## Overview

The Smart Supertext system uses 7 MOGRT templates, one per text type. Each MOGRT has a distinct visual layout optimized for its educational purpose. Font: DM Sans (synced via Adobe Fonts).

## MOGRT Files

Generated by `create-supertext-mogrt.jsx` into `~/Desktop/EditorPro_Supertexts/`:

| Filename | Type | Layout | Description |
|---|---|---|---|
| `TITLE.mogrt` | title | **text** | Big impactful text (58pt SemiBold, white). No background. For topic/section headers. |
| `BULLET.mogrt` | bullet | **text** | Simple clean line (44pt Regular, white). No background. For key points. |
| `HIGHLIGHT.mogrt` | highlight | **text** | Medium weight emphasis (50pt Medium, gold). No background. For memorable phrases. |
| `DEFINITION.mogrt` | definition | **label** | Label "DEFINICIÓN" above + text line (48pt Regular, orange). For term definitions. |
| `STEP.mogrt` | step | **card** | Dark rounded background + "PASO" badge + compact text (36pt). For procedures. |
| `DATA.mogrt` | data | **card** | Dark rounded background + "DATO" badge + compact text (36pt). For stats/URLs. |
| `SUMMARY.mogrt` | summary | **slide** | Dark background + "RESUMEN" title + body text (40pt). For section conclusions. |

## Layouts

### text (title, bullet, highlight)
Simple text on screen, lower-left positioned. No decorations. Differentiated by font weight, size, and color.

### label (definition)
Small colored label text above the main text line. Label syncs color with the Text Color control.

### card (step, data)
Dark semi-transparent rounded rectangle with a small colored badge label at top-left and content text inside. Compact Lower Third style.

### slide (summary)
Larger dark background with a colored title ("RESUMEN") at top and body text below. Designed as an end-of-section recap card.

## Loading in Editor-Pro

1. In Smart Supertext 2, click the folder button ("Carpeta")
2. Select the `EditorPro_Supertexts` folder
3. All 7 types auto-match by filename (the client's `loadMOGRTFolder` matches filenames containing each type name, case-insensitive)

## Essential Graphics Controls (per MOGRT)

Each MOGRT exposes the following controls in Premiere's Essential Graphics panel:

| Control | Type | Default | Description |
|---|---|---|---|
| **Source Text** | Text | Placeholder | Main text content. Set by `_setMGTText` in host. Supports `\r` for line breaks. |
| **Label** | Text | Type-specific | Badge text above main text (e.g., "TEMA", "PASO"). Only present on types that have labels. |
| **Text Color** | Color | Type-specific | Fill color for all text. Can be changed per clip in Premiere. |
| **Frames Per Word** | Slider | 4 | Controls IN animation speed. Higher = slower reveal. Formula: `wordCount × FPW`, clamped 15–60 frames. |
| **Position Y** | Slider | 20 | Slide distance in pixels for word-by-word entry (bottom to top). Also used as OUT slide distance. |
| **Ease High** | Slider | 100 | Per-word transition smoothness when word exits the selector (appears). Range 0–100. |
| **Ease Low** | Slider | 50 | Per-word transition smoothness when word enters the selector. Range 0–100. |
| **Out Frames** | Slider | 15 | Duration of the OUT dissolve animation in frames. |

## Animation System

### IN Animation (word-by-word)
- Text Animator with Position Y offset + Opacity 0
- Range Selector: Based On Words, Shape Square, Ease High/Low linked to sliders
- Expression on Start property counts words from Source Text and calculates duration:
  - `animFrames = clamp(wordCount × Frames Per Word, 15, 60)`
  - Uses `ease()` from `inPoint` to `inPoint + animSec`
  - Constant speed per word regardless of text length

### HOLD
- Start at 100% = all text visible, no animator effect
- Layer opacity at 100%

### OUT Animation (all-at-once dissolve)
- NOT word-by-word — entire text fades and slides simultaneously
- Layer Opacity: eases from 100 to 0 over `Out Frames`
- Layer Position: slides DOWN by `Position Y` pixels (same direction as entry)
- All layers (text, label, background) share the same OUT timing
- Card/slide backgrounds also fade and slide with the text

## Compatibility with Host Pipeline

### What the host sets automatically:
- **Source Text** via `_setMGTText()` — finds the property by display name containing "Text"/"Source"
- **Duration** via `_setMogrtClipEnd()` — trims the timeline clip
- **Position Y** (Motion component) via `_setClipPositionY()` — for bullet stacking (`bulletPositionY`)
- **Clip name** — `[TYPE] text preview`

### What the host does NOT set (manual or future):
- Text Color — uses the MOGRT's default per type
- Frames Per Word, Ease values — uses defaults
- Label text — uses the MOGRT's built-in label
- Out Frames — uses default

### Bullet Stacking
When the AI detects a list (title + bullets), the system:
1. Places each item on a separate video track (`bulletTrackOffset`: 0, 1, 2, ...)
2. Offsets Y position using `bulletPositionY` (negative = moves up)
3. Spacing: `ST2_BULLET_SPACING = 70px` between items
4. Title gets extra spacing: `titleSpacing = ST2_BULLET_SPACING × 0.4`

The MOGRT's default position is lower-left (margin ~100px from left, ~72% down). Stacking moves clips upward via Premiere's Motion > Position Y.

### Timing
- `ST2_ANTICIPATION_SECS = 1.0` — graphics appear 1s before the narrator says the concept
- `ST2_READING_BUFFER_SECS = 1.5` — extra time after narrator finishes
- `ST2_MIN_DURATION_SECS = 3` — minimum clip duration
- `ST2_SECS_PER_WORD = 0.25` — reading time calculation

## Extending the System

### To change colors per type automatically:
Extend `insertSupertextMOGRTs` in `host/index.jsx` to set the "Text Color" Essential Graphics property after inserting each clip. The property is a Color Control accessible via `getMGTComponent()`.

### To add new types:
1. Add the type name to `ST2_TYPES` array in `client/js/main.js`
2. Add a color to `ST2_TYPE_COLORS`
3. Add to `state.mogrtPaths`
4. Create the MOGRT (add to TYPES in `create-supertext-mogrt.jsx`)
5. Update the AI prompt to recognize the new type

### To change the visual design:
Run `create-supertext-mogrt.jsx` in After Effects after modifying the TYPES configuration at the top of the script. Each type's color, font, font size, and label are configurable there.
